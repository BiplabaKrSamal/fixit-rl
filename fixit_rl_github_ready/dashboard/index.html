Estimator Assumptions & Failure Modes

Method: Clipped Per-Decision Importance Sampling (PDIS)

① Common support: eval policy must not take actions never taken by the behaviour policy. Violated when RL explores SUMMARISE/RECOVER which the scripted bot never uses → clipping kicks in → biased estimate.


② No hidden confounders: logged state must fully explain the behaviour policy's choices. Real bots may have account history, caller ID → unmeasured confounding → biased rewards.


③ Stationarity: the conversation distribution doesn't shift between logging and evaluation. This is structurally violated the moment the new policy is deployed.


⚠ When it gives the wrong answer: If frustrated callers hang up before CSAT (informative dropout), logged CSAT is a biased upward sample. IPS then overestimates value for any policy, including a worse one.
